## 实现方案：时间轴页面一键创建 alias 到 NAS

### 需求
在 `/tmp/quota-timeline.html` 页面上，选一个 key，点"创建到 NAS"，自动把这个 key 注册成 alias，`expires` 设为它的周额度重置时间（提前 1 分钟）。之后你在 NAS 管理界面设分组即可。

### 需要改两个地方

---

#### 改动 1：Go 代理 — 加 JSON 版快速创建 API + CORS

**文件：`admin.go`**

1. **新增 `POST /__admin/keys/quickadd`**（JSON 版，带 CORS）
   - 接收 JSON：`{"alias": "蒋志杰", "key": "xxx.yyy", "expires": "2026-07-17 17:59"}`
   - `header` 默认空（自动检测），`rate/burst` 默认 0
   - 复用现有 `a.keys.setConfig()` 持久化到 keys.yaml
   - 返回 JSON：`{"ok": true}` 或 `{"ok": false, "error": "..."}`
   - **不走 `requireAuth`，改用独立鉴权**（见下）

2. **新增 CORS 中间件**（只对 quickadd 端点）
   - 响应头加 `Access-Control-Allow-Origin: *`
   - 处理 `OPTIONS` 预检请求
   - 允许 `POST` + `Content-Type: application/json`

3. **鉴权方式**：因为 CORS + `SameSite=Lax` cookie 跨域带不上，改用 **header token 认证**
   - 请求带 `X-Admin-Password: <密码>` header
   - 服务端用 `subtle.ConstantTimeCompare` 对比 `a.password`
   - 比搞 cookie 简单，且不受 CORS/cookie 策略限制

**路由注册**（admin.go:165 后面加一行）：
```go
mux.HandleFunc("/__admin/keys/quickadd", a.handleKeyQuickAdd)
```

**handleKeyQuickAdd 大致逻辑**：
```go
func (a *adminServer) handleKeyQuickAdd(w http.ResponseWriter, r *http.Request) {
    // CORS
    w.Header().Set("Access-Control-Allow-Origin", "*")
    w.Header().Set("Access-Control-Allow-Headers", "Content-Type, X-Admin-Password")
    w.Header().Set("Access-Control-Allow-Methods", "POST, OPTIONS")
    if r.Method == "OPTIONS" { w.WriteHeader(204); return }
    if r.Method != "POST" { jsonError(w, "method not allowed"); return }

    // token 鉴权
    pw := r.Header.Get("X-Admin-Password")
    if subtle.ConstantTimeCompare([]byte(pw), []byte(a.password)) != 1 {
        jsonError(w, "认证失败", 401); return
    }

    // 解析 JSON body
    var req struct {
        Alias   string `json:"alias"`
        Key     string `json:"key"`
        Expires string `json:"expires"`  // 可选
    }
    json.NewDecoder(r.Body).Decode(&req)
    // 校验 + setConfig
    cfg := KeyConfig{Key: req.Key, Expires: req.Expires}
    a.keys.setConfig(req.Alias, cfg)
    jsonOK(w)
}
```

---

#### 改动 2：HTML 页面 — 加"创建到 NAS"按钮 + 配置区

**文件：`/tmp/quota-timeline.html`**

1. **顶部加 NAS 配置区**（输入 NAS 地址 + 管理密码，存 localStorage）：
   ```
   NAS 地址: [https://p.19930810.xyz:8443]  管理密码: [********]  [保存]
   ```

2. **每行加"➕创建"按钮**：
   - 点击后弹出确认，显示 alias + expires
   - 自动计算 expires = 周额度重置时间 - 1 分钟
   - POST 到 `{NAS地址}/__admin/keys/quickadd`
   - 成功后按钮变 ✓

3. **已满区域也支持创建**（重置后就能用）

---

### 流程
```
时间轴页面
  → 选一个 key（比如蒋志杰，周额度今天18:00重置）
  → 点"➕创建到NAS"
  → 页面自动算 expires = 17:59
  → POST /__admin/keys/quickadd {alias:"蒋志杰", key:"xxx", expires:"2026-07-17 17:59"}
  → NAS 持久化到 keys.yaml
  → 你去 NAS 管理界面把"蒋志杰"加入分组
  → 完成
```

### 部署
改完 Go 代码后，commit + tag v2.5.19 + 推送 → CI 自动部署 jd → 手动触发 NAS。

### 风险
- 管理密码存在本地 localStorage，只在浏览器本地，不上传第三方。但 `file://` 打开的 HTML 本身就是你本机，安全。
- CORS 只对 quickadd 一个端点开放，其他 admin 路由不受影响。
- token 认证用 `ConstantTimeCompare`，防时序攻击。

### 不改的东西
- 不改现有 `handleKeyNew`（表单版保持原样）
- 不改 cookie 机制（其他 admin 页面照常）
- 不改 time_block（用 expires 代替，语义更贴合）
