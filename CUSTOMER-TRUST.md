# llm-http-proxy 客户信任说明

> 如果你正在使用经本代理转发的 API 服务，本文档帮助你确认：**这个代理不会偷取或泄露你的数据**。

---

## 一句话保证

本代理是**百分百透明转发**：你的请求（含 API key、请求体）原样转给目标 API，代理**不记录明文 key、不记录请求内容、不向外发送任何额外数据**。代码完全开源，可自行审查。

---

## 四项保证

### 1. 代码完全开源

- 仓库：**https://github.com/dyyz1993/llm-http-proxy**
- 任何人都可以 `git clone` 阅读全部代码，没有任何闭源/二进制黑盒
- 所有逻辑（转发、统计、日志）都在 `main.go` 和 `stats.go` 两个文件里，总共不到 1000 行

### 2. 不记录你的敏感数据

代理只记录以下**脱敏**信息用于统计：

| 记录什么 | 怎么处理 | 举例 |
|---------|---------|------|
| API key | **掩码**（保留前缀+后4位，中间用 `*`） | `sk-abcd...wxyz` → `sk-****wxyz` |
| 来源 IP | 直接记录（统计必需） | `203.0.113.5` |
| 目标域名 | 只记 host，**不记 path** | `open.bigmodel.cn` |
| 状态码 | 记录 | `200` / `500` |

**绝对不记录**：
- ❌ 明文 API key（掩码后也还原不了）
- ❌ 请求 body（你的 prompt 内容、对话）
- ❌ 完整 URL 路径（只记域名）
- ❌ 完整请求头（除掩码 key 外）

### 3. 不向外发送额外数据

代理**只做一件事**：把你的请求转发到你指定的目标 API。

- 没有"上报"、"心跳"、"统计回传"到任何第三方
- 没有内置的外联地址
- 代码里所有网络调用都是"转发你主动发的请求"，不存在主动外联

你可以自己验证（见下一节）。

### 4. 转发是透明的（目标 API 看不出经过代理）

代理会自动**剥离反代指纹头**（`X-Forwarded-*`、`Via`、`CF-*` 等），让目标 API 收到的请求和你直连时**完全一致**。目标 API 无法从请求特征判断出经过了代理。

## 如何自己验证

如果你不信任上述声明，可以自己查证：

```bash
# 1. 克隆代码,逐行阅读
git clone https://github.com/dyyz1993/llm-http-proxy
cd llm-http-proxy
# 核心逻辑在 main.go(转发)和 stats.go(统计/日志)

# 2. 搜"有没有偷传数据"的代码
# 搜所有网络调用,确认只有转发逻辑,无外联
grep -rn "http.Get\|http.Post\|http.NewRequest" *.go | grep -v _test.go
# 预期:只有 NewRequestWithContext(你的请求转发),没有向固定地址的上报

# 3. 搜"key 怎么处理"
grep -n "maskKey\|extractKey" stats.go
# 预期:key 提取后立即掩码,只在掩码后存储

# 4. 看日志格式(确认不记 body)
grep -n "log.Printf\|logRequest" *.go
# 预期:日志只有 ip=掩码key=host=status=,没有 body
```

更详细的审查清单见 [SECURITY-AUDIT.md](SECURITY-AUDIT.md)（给安全审查者用）。

## 总结

| 问题 | 答案 |
|------|------|
| 代理会偷我的 API key 吗？ | 不会。key 只用于掩码统计，掩码后无法还原 |
| 代理会记录我的对话内容吗？ | 不会。请求 body 原样转发，不记录 |
| 代理会向外发数据吗？ | 不会。只有你主动发的请求被转发 |
| 目标 API 会发现我用了代理吗？ | 不会。反代指纹头被剥离，请求看起来像直连 |
| 代码可信吗？ | 完全开源，可逐行审查，可本地复现构建 |

如有疑问，可在 https://github.com/dyyz1993/llm-http-proxy/issues 提问。
